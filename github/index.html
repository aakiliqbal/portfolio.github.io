What are the git directory and the working tree?

The git directory contains all the changes and their history and the working tree contains the current versions of the files.

With that, we've added our file to the staging area. The staging area which is also known as the index.

5.Advanced Git Interaction

command->

git commit -a

  • This flag automatically stages every file that's tracked and modified before doing the commit letting it skip the git add step.
  • git commit -a doesn't work on new files because those are untracked.


We'll also use the -m flag to add the commit message directly.

command ->
git log -p

  • to look at the actual lines that changed in each commit.
  • where -p comes from patch.
  • The format is equivalent to the diff-u output.




command->
git show

This command takes a commit ID as a parameter, and will display the information about the commit and the associated patch.


command->
git log --stat

This will cause git log to show some stats about the changes in the commit, like which files were changed and how many lines were added or removed.


command ->
git diff

  • To help us keep track git gives us the git diff command.
  • Again, this format is equivalent to the diff -u output


command ->
git add -p

When we use this flag, git will show us the change being added and ask us if we want to stage it or not.



command->
git diff --staged

If we call git diff again, it won't show any differences, since git diff shows only unstaged changes by default. Instead, we can call git diff -- staged to see the changes that are staged but not committed.

6.Undoing Things -

command->
git checkout

We can change a file back to its earlier committed state by using the git checkout command followed by the name of the file you want to revert.
Note - this command works if the file is not in staging state.


command->
git reset

If we realize we've added something to the staging area that we didn't actually want to commit, we can unstage our changes by using the git reset command.

git add*

This command will end up adding any change done in the working tree to the staging area.
where the star is a file glob pattern used in Bash that expands to all files.


Amending Commits- 
if there is error in our commit to amend this , we use below written command.

command->

Rollbacks-

command->
git revert

git revert doesn't just mean undo. Instead, it creates a commit that contains the inverse of all the changes made in the bad commit in order to cancel them out.




Cheat sheet-

git checkout is effectively used to switch branches.
git reset basically resets the repo, throwing away some changes. It’s somewhat difficult to understand, so reading the examples in the documentation may be a bit more useful.
There are some other useful articles online, which discuss more aggressive approaches to resetting the repo.
git commit --amend is used to make changes to commits after-the-fact, which can be useful for making notes about a given commit.
git revert makes a new commit which effectively rolls back a previous commit. It’s a bit like an undo command.
There are a few ways you can rollback commits in Git.
There are some interesting considerations about how git object data is stored, such as the usage of sha-1.
7.Branching And Merging-

So what is a branch?

What is it used for?

  • In Git, a branch at the most basic level is just a pointer to a particular commit. 
  • But more importantly, it represents an independent line of development in a project.
  • Of which the commit it points to is the latest link in a chain of developing history. 
  • The default branch that Git creates for you when a new repository initialized is called master.

Creating New Branches-

command->

  • git branch

            list the branches in current repository

  • git branch name_of_the_new_branch

            creates a new branch

  • git checkout branch_name

            for selecting any branch

  • git checkout -b branch_name

            for creating as well as selecting that new branch


  • git branch -d branch_name

            To delete a branch


Merging-

Merging is the term that Git uses for combining branch data and history together.


command ->

git merge branch_name



This is an example of fast-forward merge.
Git uses two different algorithms to perform a merge, fast-forward and three-way merge.

fast-forward merge-

This kind of merge occurs when all the commits in the checked out branch are also in the branch that's being merged.





Three-way merge-

a three-way merge is performed when the history of the merging branches has diverged in some way, and there isn't a nice linear path to combine them via fast-forwarding.


Merge Conflict-

When, we might find that both the branches we're trying to merge have edits to the same part of the same file. This will result in something called a merge conflict.


we have to solve the conflicts by editing the file which is latest.

after removing the conflict we can use this command for better representation of merge.

command->

git log --graph --oneline




command->

git merge --abort

This will stop the merge and reset the files in your working tree back to the previous commit before the merge ever happened.
